### server.js — Бэкенд (264 строки)

- **Запуск:** `node server.js`, порт 3000
- **Режимы:** чистый `http.createServer`, без Express
- **Роуты:**
  - `/*` — раздача статики (index.html, .css, .js, .svg) через `fs.readFileSync`
  - `POST /generate-haiku`, `/generateHaiku`, `/generate` — генерация хайку через OpenAI (с rate limiting: 10 запросов/IP/мин)
  - `GET /history` — возврат истории генераций (in-memory массив, без ограничения длины)
- **OpenAI API:** ручной HTTPS-запрос, `JSON.parse` для разбора ответа OpenAI
  - Обычный режим: `gpt-4o-mini`
  - При любом wasabiLevel > 0: `gpt-4` (уровень клиента 0-6, но сервер капит на 3)
- **Особенности:** искусственная задержка 1500 мс «для солидности»

### index.html — Фронтенд-структура (87 строк)

- 6 секций-карточек + сплэш-скрин
- Карточки: результат, ключевые слова, язык, васаби, история, информация (с кнопкой генерации внутри)
- Подключение Google Fonts, скриптов, стилей

### script.js — Клиентская логика (~500 строк)

- **Состояние:** единый объект `state` с полями `words`, `language`, `wasabiLevel`, `result`, `lines`, `history`
- **Рендеринг:** универсальная функция `render()`, вызывающая частные `render*()` по секциям
- **Генерация:**
  1. `generate()` — валидация (3-7 слов, язык), отправка POST на сервер
  2. При ошибке — `offlineHaiku()` — процедурная генерация по ключевым словам (набор заготовок-джуайнеров, хвостов, модификаторов)
  3. При наличии `window.claude.complete()` — использует API Claude встроенного терминала
- **История:** `localStorage` с ключом `haiku50_history`, до 8 записей
- **Сплэш:** 3-секундная анимация при загрузке

### styles.css — Стилизация (854 строки)

- `backdrop-filter: blur()` для эффекта стекла
- Три брейкпоинта: >979px (3 колонки), 640-979px (2 колонки), <640px (1 колонка)
- Анимации: спин, fade-up, pop, сплэш
- Цветовая схема: кремовый фон (#f1efe8), зелёный акцент (#5c8a4a), тёмный текст (#232220)

## Точки входа

### Запуск приложения
```bash
node haiku-50/server.js
```
→ Открыть `http://localhost:3000` в браузере.

### Поток запроса генерации
```
Браузер → POST /generate-haiku {words, language, wasabiLevel}
  → server.js: парсинг тела → выбор модели (GPT-4o-mini/GPT-4)
  → server.js: HTTPS-запрос к api.openai.com/v1/chat/completions
  → server.js: JSON.parse ответа OpenAI → ответ клиенту
  → script.js: рендеринг результата → сохранение в историю
```

### Офлайн-поток (fallback)
```
POST /generate-haiku (ошибка сети или сервера)
  → script.js: catch → offlineHaiku()
  → алгоритмическая сборка хайку из ключевых слов + заготовок
  → рендеринг результата
```

## Runtime-потоки

### Поток 1: Инициализация SPA
```
1. Загрузка index.html
2. Парсинг CSS, загрузка Google Fonts
3. Исполнение script.js:
   a. Инициализация state (пустые значения)
   b. Загрузка истории из localStorage
   c. Рендеринг всех секций (render())
   d. Запуск сплэш-скрина (3 сек)
```

### Поток 2: Генерация хайку (основной)
```
1. Пользователь вводит 3-7 ключевых слов
2. Выбирает язык (фактически только Japanese)
3. Выбирает уровень васаби (0-6)
4. Нажимает «Generate haiku»
5. script.js:
   a. Валидация (есть ли слова? выбран ли язык?)
   b. Установка state.result = 'loading'
   c. POST /generate-haiku с body {words, language, wasabiLevel}
6. server.js:
   a. Чтение тела запроса
   b. Выбор модели по wasabiLevel
   c. Формирование prompt для OpenAI
   d. HTTPS-запрос к api.openai.com (1500ms задержка)
   e. Парсинг ответа через JSON.parse
   f. Возврат JSON {success, lines, original, source}
   g. Сохранение в in-memory history
7. script.js:
   a. Проверка success
   b. Обновление state с линиями хайку
   c. render() → отображение результата
   d. Добавление в историю → localStorage
```

### Поток 3: Генерация хайку (офлайн-fallback)
```
1-4. Те же шаги
5a. POST /generate-haiku → ошибка (сеть/сервер/OpenAI)
5b. script.js catch → offlineHaiku(state.words, state.wasabiLevel)
6. offlineHaiku():
   a. Фиксированная структура: line1 / line2 / line3
   b. Ротация ключевых слов
   c. Вставка joiners, tails, temperature-модификаторов
   d. Возврат массива строк
7. script.js: рендеринг результата
```

### Поток 4: Обработка ошибок
```
Ошибка OpenAI (квота, таймаут, невалидный JSON):
  → server.js: возврат {success: false, error: "..."}
  → script.js: рендеринг ошибки, запуск offlineHaiku()

Общая ошибка сервера:
  → server.js: process.on('uncaughtException') → console.error со stack trace + process.exit(1)
  → Клиент: fetch упадёт с сетевой ошибкой
  → script.js: offlineHaiku()

Невалидный JSON от OpenAI:
  → server.js: try/catch → JSON.parse → fallback на makeHaikuLocal()
  → Если ответ не содержит choices[0].message.content → возврат ошибки
```

## Зоны повышенного риска

### 1. 🔴 Хардкод API-ключа OpenAI
**Файл:** `haiku-50/server.js:11`
```javascript
const apiKey = "sk-proj-..."; // ключ был хардкожен, позже вынесен в .env
```
- Ключ в открытом виде в репозитории
- Нет `.env` файла, нет переменных окружения
- **Риск:** компрометация ключа, финансовые потери

### 2. 🟡 Парсинг тела POST-запроса через регулярные выражения
**Файл:** `haiku-50/server.js:170-180`
- Входящий POST-запрос от браузера разбирается ручным regex вместо `JSON.parse`
- Ответ OpenAI при этом парсится через `JSON.parse` (строки 84, 130) — корректно
- **Риск (regex парсинг тела):** нестабильность при нестандартном форматировании JSON, потеря данных, потенциальные уязвимости

### 3. 🟡 Искусственная задержка 1500ms
**Файл:** `haiku-50/server.js:163`
- «Чтобы выглядело солиднее»
- **Риск:** ухудшение UX, таймауты при медленном соединении

### 4. ✅ Глушение всех исключений — ИСПРАВЛЕНО
**Файл:** `haiku-50/server.js:20-24`
```javascript
process.on("uncaughtException", function (err) {
  console.error("❌ Непойманная ошибка:", err);
  console.error("Stack trace:", err && err.stack);
  process.exit(1);
});
```
- Ошибка логируется с полным stack trace в stderr
- Процесс завершается, не продолжая работу в неопределённом состоянии
- **Статус:** исправлено — добавлен console.error + process.exit(1)

### 5. 🟡 Нет package.json
- Не фиксирована версия Node.js
- Нет списка зависимостей
- **Риск:** невоспроизводимость окружения, неясно, какие модули используются

### 6. 🟢 Нет обработки лимитов OpenAI
- Нет проверки квоты, нет ретраев с экспоненциальной задержкой
- При превышении лимита API вернёт 429, пользователь увидит ошибку

### 7. 🟢 Нет rate limiting на сервере → ✅ ИСПРАВЛЕНО
- Добавлен in-memory sliding window (10 запросов/IP/минуту)
- Возвращает HTTP 429 при превышении
- Защищает от финансовых потерь при злоупотреблении OpenAI API
- **Файл:** `server.js:20-23, 44-61, 195-197`

### 8. 🟢 Синхронное чтение файлов
```javascript
let html = fs.readFileSync("index.html", "utf8");
```
- Блокирует event loop при каждом запросе статики
- **Риск:** деградация производительности под нагрузкой

### 9. 🟢 Офлайн-генератор — не тестирован
- `offlineHaiku()` — единственный fallback при отказе OpenAI
- Не покрыт тестами, работает «как получится»
- **Риск:** неожиданное поведение при сбоях
